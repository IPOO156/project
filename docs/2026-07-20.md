## 二、根因（4 层嵌套）

### 第 1 层：用户系统开启"减少动画"

Windows 设置 → 辅助功能 → 视觉 → 动画效果 = 开启。
浏览器在 `prefers-reduced-motion: reduce` 媒体查询匹配时，会**强制让所有 CSS 动画/过渡直接进入终态**（即使子组件没有写兜底，浏览器 UA 行为也会抑制动画）。

### 第 2 层：2470fd0 拆分时丢失动画

| commit            | 改动                                                                                                                                                                                                                                                                             |
| ----------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `5c59354` (07-08) | TimelineNode 包含完整动画（`@keyframes gt-card-enter`、卡片进入、圈动画、进度条 stagger）                                                                                                                                                                                        |
| `2470fd0` (07-10) | **拆分时把卡片数据/模板/部分样式移到子组件 `GrowthCard.vue`，但 `@keyframes` 和动画 CSS 仍留在父级** —— **子组件的进入动画** (`opacity: 0 → 1` + `translateY(34px) → 0` + `scale(0.96) → 1`) **没有被父级 `:deep()` 触发**（父级有动画但子组件初始态不对，外部 hover 不依赖 IO） |
| `1c87164` (07-12) | 试图"修复"动画归属，但反而**删了 GrowthCard 内原本的 hover transition**，并加了一堆冗余 CSS                                                                                                                                                                                      |

**结果**：从 `2470fd0` 拆分那一刻起，**卡片进入动画其实就没真正工作过**（只在父级 TimelineNode 内部对 ring-marker 起效，对子组件 GrowthCard 没真正穿透）。

### 第 3 层：Vue scoped CSS 给 `@keyframes` 加 hash

Vue 3 编译 `<style scoped>` 时，会给 `@keyframes` 名加 `[data-v-hash]` 后缀（防止全局污染）：

```css
/* 子组件 TimelineNode.vue 里写 */
@keyframes gt-card-enter { ... }

/* 编译后 */
@keyframes gt-card-enter-c30368ed { ... }
```

外部 CSS（motion-override.scss）**无法引用** `gt-card-enter-c30368ed`（hash 是构建时随机生成的）。
导致即使 motion-override 想覆盖，也找不到 keyframe。

### 第 4 层：motion-override 错误地"加载即播"

第一版 motion-override 写：

```scss
@media (prefers-reduced-motion: reduce) {
  :root[data-allow-motion='true'] .growth-card {
    animation: gt-card-enter 0.85s... both !important; /* ❌ 不依赖 visible class */
  }
}
```

**后果**：attr=true 时，**所有 5 张卡片加载即播动画**（不管 IntersectionObserver 是否触发）。
0.85s 后动画结束，opacity=1 → **用户感知不到动画**（已经在用户没看到时播完了）。

---

## 三、解决思路

**5c59354 原始动画 + motion-override 全局强制启用**。

- **子组件**的 `prefers-reduced-motion` 兜底**全部删除**（统一管理）
- **关键帧 `gt-card-enter` 定义在全局**（`motion-override.scss`），避免 Vue scoped hash
- **`main.ts` 入口无条件设置 `<html data-allow-motion="true">`**
- **motion-override 在 `prefers-reduced-motion: reduce` 媒体查询内**用 `animation` 简写配合 `!important` 强制覆盖浏览器 UA 抑制

## 四、具体改动

### 4.1 新增文件

#### [motion-override.scss](file:///d:/code/project/src/assets/styles/motion-override.scss)

全局动画规则集中管理：

1. **全局关键帧定义**（无 hash，所有组件可引用）：

   ```scss
   @keyframes gt-card-enter {
     from {
       opacity: 0;
       transform: translateY(34px) scale(0.96);
     }
     to {
       opacity: 1;
       transform: translateY(0) scale(1);
     }
   }
   ```

2. **强制覆盖**（在 `prefers-reduced-motion: reduce` 媒体查询内）：
   ```scss
   @media (prefers-reduced-motion: reduce) {
     .growth-node.visible .growth-card,
     .growth-node.visible .ring-marker {
       animation: gt-card-enter 0.85s cubic-bezier(0.16, 1, 0.3, 1) var(--stagger-delay, 0s) both !important;
     }
   }
   ```

**关键约束**：

- ❌ **不能**写 `opacity: 0 !important` 或 `transform: ... !important` —— 这会**锁死** keyframe 的 `to` 帧
- ❌ **不能**用 `animation-duration: revert` 关键字 —— 会回退到 initial 0s
- ✅ 必须用 `animation` 简写一次性覆盖 `name + duration + delay + fill-mode`
- ✅ **必须依赖 `.visible` class** —— 加载即播会导致用户在没看到时动画就播完了

## 七、经验教训

1. **不要拆分组件时丢失 CSS** —— 模板和数据可以拆，但**关键动画 CSS 必须保留**。拆分前先在 5c59354 跑一遍动画，拆分后立刻验证。
2. **Vue scoped CSS 的 `@keyframes` 会被加 hash** —— 全局引用的关键帧必须放在非 scoped 的 CSS（motion-override.scss）里。
3. **优先用 `:deep()` 让父级穿透到子组件**，不要在子组件内重写动画（容易丢失父级的 IO 触发逻辑）。
4. **CSS `!important` + `animation` 简写**是绕过浏览器 `prefers-reduced-motion` UA 抑制的唯一可靠方式。
5. **不要**用 `revert` 关键字试图"恢复"动画时长 —— 会回退到 initial 0s。
6. **不要**写 `opacity: 0 !important` + `transform: ... !important` —— 会锁死 keyframe 的 to 帧。
7. **动画覆盖必须依赖 `.visible` class** —— 否则加载即播，用户感知不到。
8. **始终启用比"用户开关"更简单** —— 项目内部使用场景下，无障碍兼容不如"动画可读性"重要，硬编码策略减少了心智负担。如果未来需要给无障碍用户开关，可参照 [2026-07-20 commit](https://github.com/) 重建 useMotionToggle composable（git log 搜 `useMotionToggle`）。
