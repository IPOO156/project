# 首页快捷入口与最近动态功能评估报告

## 1. 评估范围

本报告针对以下两部分功能进行用户体验、性能、安全性及可扩展性评估：

1. **首页快捷入口优化**：在 `Dashboard.vue` 中落地的 5 项企业级优化（交互体验、视觉设计、个性化推荐、性能优化、移动端适配）。
2. **最近动态独立列表页**：新增 `/activities` 页面，支持动态记录的增删改查，且 `approved` 记录只读。

---

## 2. 用户体验（UX）评估

### 2.1 快捷入口

| 评估项   | 现状                                          | 评分       | 说明                           |

| 点击反馈 | hover 上移 + 图标放大 + 箭头滑入              | ⭐⭐⭐⭐⭐ | 多通道反馈，操作感知明确       |
| 错误处理 | `async/await + try/catch` + `ElMessage.error` | ⭐⭐⭐⭐⭐ | 跳转失败有明确提示             |
| 可访问性 | 支持 `tabindex` 与 `Enter/Space` 键盘触发     | ⭐⭐⭐⭐⭐ | 满足 WCAG 2.1 键盘操作基本要求 |
| 个性化   | 基于 localStorage 记录最近 3 次点击并排序     | ⭐⭐⭐⭐☆  | 简单有效，但无法跨设备同步     |
| 移动端   | 响应式 grid，小屏单列，触控区 ≥44px           | ⭐⭐⭐⭐⭐ | 满足移动端可用性要求           |

### 2.2 最近动态列表页

| 评估项   | 现状                                           | 评分       | 说明                     |
| -------- | ---------------------------------------------- | ---------- | ------------------------ |
| 信息架构 | 搜索 + 状态筛选 + 表格 + 分页                  | ⭐⭐⭐⭐⭐ | 与现有提交记录页模式一致 |

## 3. 性能评估

| 评估项   | 现状                                 | 风险等级 | 说明                                     |
| -------- | ------------------------------------ | -------- | ---------------------------------------- |
| 首屏加载 | 图标按需导入、无额外网络请求         | 低       | `lucide-vue-next` 组件级导入，无全量打包 |
| 渲染效率 | 动画使用 `transform` + `will-change` | 低       | 避免触发重排，GPU 加速友好               |
| 本地存储 | localStorage 每次点击写入 3 条记录   | 低       | 写入量极小，不影响性能                   |
| 数据请求 | Activity Store mock 延迟 300ms       | 中       | 真实接口联调后应接入后端分页             |
| 大数据量 | 当前 mock 15 条，前端分页            | 低       | 数据量增大后可保持前端分页或接入后端分页 |

### 3.1 性能优化建议

1. **路由懒加载**：`/activities` 页面已使用 `import()` 懒加载，符合最佳实践。
2. **非首屏数据延迟初始化**：首页最近动态可在 `requestIdleCallback` 中请求，优先保障雷达图与统计卡片渲染。
3. **缓存策略**：Activity Store 增加 TTL 缓存（如 5 分钟），减少重复进入列表页时的 mock/接口请求。

## 4. 安全性评估

| 评估项   | 现状                                       | 风险等级 | 说明                                                               |
| -------- | ------------------------------------------ | -------- | ------------------------------------------------------------------ |
| XSS 防护 | 动态内容通过 `el-table` 默认渲染，自动转义 | 低       | 未使用 `v-html` 渲染用户输入                                       |
| 权限控制 | `approved` 记录在 UI 与 Store 双层拦截     | 低       | `RecordActionButtons` 隐藏编辑/删除，`ActivityStore` 拒绝修改/删除 |
| 本地存储 | localStorage 仅存储路径字符串              | 低       | 不存储敏感信息                                                     |
| 输入校验 | 动态内容非空校验、时间必填校验             | 低       | 表单提交前已校验                                                   |

### 4.1 安全优化建议

1. **后端权限校验**：前端拦截仅作为体验优化，真实删除/修改接口必须再次校验 `canEdit`/`canDelete`。
2. **内容长度限制**：动态内容输入应限制最大长度（如 200 字符），防止异常数据。
3. **CSRF 防护**：接口联调后确保使用 Cookie/Token 等防 CSRF 机制。

## 5. 可扩展性评估

| 评估项        | 现状                                                                     | 说明                                               |
| ------------- | ------------------------------------------------------------------------ | -------------------------------------------------- |
| Activity 类型 | 已独立定义 `Activity` 接口                                               | 后续可扩展 `module`、`recordId`、`operator` 等字段 |
| 快捷入口配置  | `useQuickEntries` 中 `baseEntries` 集中维护                              | 后续可从后端或配置文件读取                         |
| 推荐算法      | 当前为 localStorage 最近使用                                             | 后续可替换为服务端推荐或权重算法                   |
| 状态字典      | 复用 `APPLICATION_STATUS` 与 `ACTIVITY_STATUS`                           | 新增状态只需修改 `dict.ts`                         |
| 页面组件      | 复用 `PageContainer`、`PageHeader`、`PageToolbar`、`RecordActionButtons` | 新增列表页开发成本低                               |

### 5.1 可扩展性优化建议

1. **配置化快捷入口**：将 `baseEntries` 提取为 JSON 配置或后端接口，支持运营调整。
2. **动态事件溯源**：`Activity` 未来可由提交记录、审批记录自动产生，减少人工维护。
3. **权限字段细化**：由后端返回 `canEdit`/`canDelete` 替代单一 `status` 判断，支持更细粒度权限。

## 6. 额外优化建议（3+ 项）

### 6.1 加载速度优化

- **首屏关键路径优化**：将首页统计卡片、快捷入口作为首屏关键内容优先渲染；最近动态等非关键数据使用 `requestIdleCallback` 延迟请求。
- **接口聚合**：真实接口联调后，将首页多个独立请求聚合为一个 `dashboard/overview` 接口，减少网络往返。
- **资源预加载**：对高频跳转页面（如 `/applications`、`/growth-timeline`）使用 `<link rel="prefetch">` 在浏览器空闲时预加载。

### 6.2 移动端适配优化

- **表格卡片化**：动态列表页在 <768px 时，将 `el-table` 切换为卡片列表，避免表格横向滚动。
- **筛选抽屉**：小屏下将搜索与筛选折叠为底部或侧边抽屉，释放横向空间。
- **触控优化**：确保所有按钮、链接、下拉框触控区域 ≥44×44px，符合 WCAG 2.5.5 建议。

### 6.3 数据缓存策略

- **Store 级 TTL 缓存**：在 `useActivityStore` 中记录上次请求时间，5 分钟内重复进入 `/activities` 直接返回缓存数据。
- **本地草稿缓存**：新增/编辑动态时，若用户意外刷新，可从 `sessionStorage` 恢复未提交内容。
- **离线提示**：网络异常时展示本地缓存数据并提示「当前为离线数据」。

### 6.4 权限控制优化

- **后端权威校验**：所有删除/修改请求以服务端返回的 `canEdit`/`canDelete` 字段为准。
- **操作审计**：关键操作（删除动态、修改 approved 记录尝试）记录操作日志，便于追溯。
- **角色权限**：未来支持管理员/学生不同角色，管理员可维护全部动态，学生仅维护自己创建的动态。

---

## 7. 结论

本次优化在用户体验、性能、安全性和可扩展性方面均有明确提升：

- **快捷入口**通过视觉、交互、个性化、性能、移动端 5 个维度达到了企业级可用标准。
- **最近动态列表页**通过独立 Store、标准化组件、双层只读校验实现了安全且可维护的 CRUD。
- 后续建议优先落地 **数据缓存策略**、**移动端表格卡片化** 和 **后端权限字段细化**，以进一步提升系统健壮性。
