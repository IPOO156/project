## 二、问题 1：成长时间轴卡片淡入淡出动画消失

### 现象

- 路由 `/profile/career-plan` 的"成长记录" tab 完全没有进入动画。
- 独立的 `/growth-timeline` 路由（HeroRings + TimelineNode）动画正常。

### 根因

- 之前排查误把问题指向 `GrowthCard.vue`（独立路由的年轮卡片），**实际渲染的是 `[GrowthRecords.vue](file:///d:/code/project/src/features/profile/components/GrowthRecords.vue)`** —— 一个纯静态的竖向时间列表，**零动画**。
- `useScrollReveal.ts` 也只服务于 HeroRings/TimelineNode，不作用于 GrowthRecords。

### 修复

[GrowthRecords.vue](file:///d:/code/project/src/features/profile/components/GrowthRecords.vue)：

- 引入 `IntersectionObserver`，给每张卡片 `.growth-item` 添加 `.visible` 类。
- CSS 添加 `opacity: 0 → 1`、`translateY(14px) → 0` 的 0.5s 过渡 + 阶梯式 `transitionDelay`（最多前 6 张错峰 60ms）。
- 支持 `prefers-reduced-motion`：默认全部可见，无动画。
- 组件卸载时 `observer.disconnect()`，避免内存泄漏。

### 验收

- 滚动到对应卡片时由下向上淡入。
- 切换到"成长记录" tab 时可见首屏卡片动画播放。
- 浏览器开启"减少动态效果"时直接全部显示。

## 三、问题 2：AI 深度分析后职业规划页没有渲染分析结果

### 现象

- 点"开始分析" → AI 助手弹窗回复富文本（含"已同步至职业规划"提示）。
- 但右侧"AI 深度分析"卡片**始终是空态**（"点击「开始分析」获取个性化改进建议"）。

### 根因

模板 [CareerPlan.vue:129-130](file:///d:/code/project/src/features/profile/CareerPlan.vue#L129-L130) 把 `aiAnalysis` 当成数组遍历：

```vue
<div v-if="aiAnalysis && Array.isArray(aiAnalysis)" class="ai-result">
  <div v-for="(item, i) in aiAnalysis" :key="i" class="ai-item">
```

但 `[types.ts#CareerAnalysis](file:///d:/code/project/src/shared/types/types.ts)` 的实际结构是：

```ts
{ greeting: string; summary: string; weaknesses: WeaknessItem[]; generatedAt: string }
```

`aiAnalysis` 是个**对象**，不是数组。`Array.isArray(aiAnalysis)` 永远 `false` → 整个 `v-if` 永远不渲染。

### 修复

[CareerPlan.vue](file:///d:/code/project/src/features/profile/CareerPlan.vue) 模板 + CSS 双重改造：

- `v-for` 改为 `aiAnalysis.weaknesses`，遍历真正的短板数组。
- 新增头部（图标 + "个人化短板分析" + 生成时间）。
- 新增 summary 概览卡片。
- 每条短板显示：维度名、当前分/目标分/差距、薄弱描述、改进建议。
- 样式升级：白橙渐变 + 左侧色条 + 多层级字号。

### 验收

- 点"开始分析"后，右侧立即出现个性化短板分析卡片。
- 关闭 AI 助手弹窗后分析结果**仍保留**（不依赖弹窗生命周期）。
- 再次点"开始分析"会**刷新**最新分析结果。

### 数据流（完整链路）

1. 用户点击"开始分析" → `openAIDrawer()` 弹 AIChatDrawer
2. AIChatDrawer watch(initialQuestion) → `useAIChat().sendMessage("请分析我的职业规划短板并给出改进建议")`
3. `apiSendMessage` → `simulateAIReply` → `matchKnowledge` 命中"职业规划"
4. `analyzeCareer(dimensions, awards, grades, plans)` 生成 `CareerAnalysis`
   - 若 archiveStore.dimensions 为空 → 先 await `archiveStore.fetchArchive()` 再重试（已修于 07-10）
5. `careerPlanStore.setAIAnalysis(analysis)` 写入 store
6. `aiAnalysis = computed(() => careerPlanStore.aiAnalysis)` 响应式更新
7. 模板 `v-for="(item, i) in aiAnalysis.weaknesses"` 渲染右侧卡片

## 五、问题 4：AI 助手弹窗 X 点击无效

### 现象

- 点 AI 助手右上角 X 按钮 / 点背景遮罩 → 弹窗不关闭。
- 点 Esc 键也无效。

### 根因

[CareerPlan.vue:174-179](file:///d:/code/project/src/features/profile/CareerPlan.vue#L174-L179) 使用了错误的 v-model 绑定：

```vue
<AIChatDrawer v-model:visible="aiDrawerVisible" :initial-question="aiInitialQuestion" />
```

但 AIChatDrawer 的 props 是 `visible: boolean`，**没有 `update:visible` emit**（只 emit `close: []`）。`v-model:visible` 完全没生效，X 触发的 `close` 事件没人监听，状态永远不变。

### 修复

改为正确的事件方式：

```vue
<AIChatDrawer
  :key="aiDrawerKey"
  :visible="aiDrawerVisible"
  :initial-question="aiInitialQuestion"
  @close="aiDrawerVisible = false"
/>
```

### 验收

- 点 X / 背景遮罩 / Esc 键 → 弹窗正常关闭。

## 七、问题 6：标签栏满 7 个后体验问题

### 现象

- 打开 7 个标签后，再点新页面 → 弹警告"已达上限" → 回退到旧页 → 用户根本没看到新页面。

### 改造

[tabs store](file:///d:/code/project/src/app/stores/tabs.ts) 与 [useTabs](file:///d:/code/project/src/shared/composables/useTabs.ts) 双重升级：

1. `MAX_TABS = 7` 改为 export 导出
2. `addTab` 返回值改为 `{ ok, evicted? }`：
   - 已存在标签 → `{ ok: true }`
   - 未超限 → 追加，返回 `{ ok: true }`
   - 超限且有非固定标签可淘汰 → 自动 `splice` 最早一个 + `push` 新标签，返回 `{ ok: true, evicted }`
   - 全是固定标签（极端情况）→ `{ ok: false }`
3. `useTabs.handleRoute` 检测到 `evicted` 时弹 ElMessage：

```
标签已达上限 7 个，已自动关闭最早打开的「X」，并打开当前「Y」
```

- 带 3.5s 防抖，避免短时间内连点多个标签时多次提示。
- 个人档案信息申报仍用单标签（`/applications` 合并 fullPath）。

### 验收

- 开满 7 个标签后，点新页面：
  1. 最早打开的可关闭标签自动消失
  2. 新页面作为最后一个标签被激活
  3. 弹"标签已达上限 7 个，已自动关闭最早打开的「X」，并打开当前「Y」"提示
- 固定标签（如首页、消息中心）永远不被淘汰。
