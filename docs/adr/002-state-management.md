# ADR-002：使用 Pinia 进行状态管理

## 状态

已采纳

## 背景

项目需要在多个页面间共享用户登录态、应用配置等信息，并支持审批/申报模块间的状态传递。

## 决策

采用 Pinia 作为全局状态管理方案，按业务域拆分 Store：

- `useUserStore`：用户登录态、用户信息。
- `useAppStore`：应用级状态（菜单展开/收起、主题等）。

## 理由

- Vue 官方推荐，与 Vue 3 组合式 API 深度集成。
- TypeScript 类型支持友好，无需额外装饰器。
- 单 Store 文件控制在 200 行以内，便于维护。

## 影响

- 禁止直接修改其他模块的 Store 状态。
- 跨模块通信优先通过公共 Store、URL 参数或服务层完成。
