## 一、删除记录

### 1.1 移除10个申报模块中重复的表单提交逻辑

以下代码在10个申报模块中各删除一份（每文件约15行，合计约150行）：

```typescript
import { ElMessage } from 'element-plus'
import { reactive, ref } from 'vue'

// ...

const form = reactive(emptyForm())
const submitting = ref(false)

function reset() {
  Object.assign(form, emptyForm())
}

function handleSubmit() {
  submitting.value = true
  setTimeout(() => {
    ElMessage.success('申报提交成功')
    reset()
    submitting.value = false
  }, 600)
}
```

**涉及文件**：

- `src/features/applications/social-practice/SocialPracticeList.vue` — 删除 `ElMessage`/`reactive`/`ref` 导入、`form`/`submitting` 状态声明、`reset`/`handleSubmit` 函数

- `src/features/applications/scholarship/ScholarshipList.vue` — 同上
- `src/features/applications/research/ResearchList.vue` — 同上
- `src/features/applications/internship/InternshipList.vue` — 同上

## 二、新增/修改记录

### 2.1 新增 `useApplicationForm` Composable

**涉及文件**：`src/shared/composables/useApplicationForm.ts`（**新增**）

**内容**：

- 定义 `ApplicationFormOptions<T>` 接口，支持 `emptyForm`、`onSubmit`（可选自定义提交函数）、`successMessage`（可选成功提示）三个配置项
- 导出 `useApplicationForm<T>` 函数，返回 `{ form, submitting, reset, handleSubmit }` 四个响应式/方法引用
- 内置重复提交防护：`handleSubmit` 开头检查 `if (submitting.value) return`
- 统一错误处理：`try-catch-finally` 包裹提交流程，失败时显示 `ElMessage.error('提交失败，请重试')` 并 `console.error` 记录原始错误，`finally` 中确保 `submitting` 状态重置
- 默认模拟提交：无 `onSubmit` 时使用 `setTimeout(600)` 模拟异步操作，行为与重构前完全一致
- 支持对接真实 API：通过 `onSubmit` 参数传入 `async (form) => { await api.submit(form) }` 即可

### 2.2 重构10个申报模块使用通用 Composable

**涉及文件**：全部 10 个申报 `*List.vue`（**修改**）

**修改内容**：

每个文件的 `<script setup>` 部分将删除的代码替换为：

```typescript
import { useApplicationForm } from '@/shared/composables/useApplicationForm'

// ... emptyForm() 定义不变 ...

const { form, submitting, reset, handleSubmit } = useApplicationForm({ emptyForm })
```

模板部分（`<template>`）完全不变，默认成功消息保持 `'申报提交成功'`，与重构前行为一致。

## 三、影响范围

- **申报模块表单提交**：10个模块的提交按钮点击行为不变（600ms 模拟延迟 + `ElMessage.success` + 表单重置），且新增重复提交防护（提交中按钮不可再次点击）。
- **表单重置**：`reset` 函数逻辑由 composable 统一提供，行为与重构前一致。
- **可扩展性**：后续如需统一修改提交行为（如添加表单验证、修改成功提示文案、对接真实 API），只需修改 `useApplicationForm.ts` 一处，全部 10 个模块自动生效。
- **模板层无变更**：`<template>` 中 `@submit="handleSubmit"`、`:submitting="submitting"`、`v-model="form.xxx"` 等绑定全部保持不变。
