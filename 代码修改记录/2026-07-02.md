### 1.1 升级 `DefaultLayout.vue` 全局布局容器

**涉及文件**：`src/app/layouts/DefaultLayout.vue`

**升级内容**：

- **多 slot 体系**：从简单双 slot（Sidebar / Main）扩展为完整 6 slot：
  - `#header` — 页面级标题 + 描述
  - `#toolbar` — 搜索/筛选/操作按钮
  - `#breadcrumb` — 面包屑（默认自动从路由 meta 推断，可覆盖）
  - `#footer` — 可选页面底栏
  - `#default`（router-view）— 主内容区
  - 全局 loading 遮罩（`appStore.pageLoading`）

- **自动面包屑**：从 `route.matched` 推断面包屑链，配合路由 `meta.title`，无需每个页面手动传入

- **移动端响应**：通过 `appStore.isMobile`（断点 `< 992px`）自动切换，Sidebar 收起，内容区 `margin-left: 0`

- **路由过渡动画**：`<transition name="layout-fade" mode="out-in">`，opacity + translateY，200ms ease

- **全局 Loading 态**：`appStore.pageLoading` 控制半透明遮罩 + 旋转 spinner，支持 `useTableQuery` 等 composable 接入

- **CSS 变量化**：所有硬编码值替换为 SCSS token（`$spacing-xl` / `$radius-lg` / `$shadow-base` 等）

**修改前后对比（核心结构）**：

```vue
<!-- 修改前：仅两个固定 slot，标题/面包屑散落在页面内部 -->
<div class="layout">
  <Sidebar />
  <div class="layout__main">
    <HeaderBar />
    <main class="layout__content">
      <router-view />
    </main>
  </div>
</div>
```

```vue
<!-- 修改后：6 slot + 自动面包屑 + 移动端 + Loading -->
<div class="layout" :class="{ 'layout--collapsed': collapsed, 'layout--mobile': isMobile }">
  <Sidebar />
  <div class="layout__main">
    <HeaderBar />
    <main class="layout__content">
      <div class="layout__content-inner">
        <!-- #breadcrumb + #header + #toolbar -->
        <router-view v-slot="{ Component }">
          <transition name="layout-fade" mode="out-in">
            <component :is="Component" />
          </transition>
        </router-view>
      </div>
    </main>
  </div>
  <!-- 移动端遮罩 + Loading 遮罩 -->
</div>
```

### 2.1 新增 `.cursor/` AI 规则体系

**涉及文件**：

- `.cursor/AGENTS.md` — 项目全局上下文（必读入口）
- `.cursor/rules/rules.md` — 规则索引文件
- `.cursor/rules/ui-ux-rules.md` — 完整 UI/UX 规范（颜色/字体/间距/动画/可访问性）
- `.cursor/rules/ui-ux-rules.mdc` — MDC 格式规则（Cursor IDE 原生支持）
- `.cursor/rules/vue-composition-rules.md` — Vue 3 + TypeScript 写法规范
- `.cursor/rules/layout-patterns.md` — 页面结构与布局模式（列表/表单/详情/仪表盘）
- `.cursor/rules/project-context.mdc` — MDC 格式项目上下文

**核心规则摘要**（10 条强制约束）：

| # | 约束 | 违反后果 |
|---|---|---|
| 1 | 禁止 emoji 作为 UI 图标，统一用 Lucide | 风格不一致 |
| 2 | 颜色全部 token（`$color-primary` 等），禁止硬编码 | 主题失效 |
| 3 | 卡片圆角统一 `$radius-lg` (8px) | 视觉不统一 |
| 4 | 阴影默认 `$shadow-base`，弹窗 `$shadow-lg` | 层次不清 |
| 5 | 页面区块间距 `$spacing-xl` (24px) | 间距混乱 |
| 6 | 过渡 150–300ms，用 `cubic-bezier(0.4, 0, 0.2, 1)` | 动画不流畅 |
| 7 | hover 反馈用 `background-color`/`box-shadow`，禁止 `scale` | layout shift |
| 8 | 图标按钮必须有 `aria-label` | 可访问性违规 |
| 9 | `focus-visible` 必须有可见轮廓 | 键盘导航失效 |
| 10 | 文字/背景对比度 ≥ 4.5:1 | 可访问性违规 |

### 2.3 新增共享 UI 组件

**涉及文件**：`src/shared/ui/` 目录

| 文件 | 组件 | 用途 |
|---|---|---|
| `PageHeader.vue` | 页面级统一标题 | 自动左侧装饰条、副标题、slot for actions |
| `PageToolbar.vue` | 页面级工具栏 | v-model 搜索/筛选、右侧操作按钮插槽 |

**使用示例**：

```vue

<template>
  <PageContainer :loading="loading" :empty="!loading && list.length === 0">
    <PageToolbar v-model:keyword="keyword" v-model:status="status">
      <el-button type="primary">新增</el-button>
    </PageToolbar>
    <el-table :data="list">...</el-table>
  </PageContainer>
</template>
```
