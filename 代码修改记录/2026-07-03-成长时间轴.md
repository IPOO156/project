2. 悬停电子时放大并高亮。
3. 点击电子：相机平滑移动到电子位置，经历卡片从电子处由小变大铺满全屏，展示标题、学期、日期、描述、标签、能力成长条。
4. 点击原子核：弹出表单，填写标题、日期、描述、标签、能力成长，提交后根据日期自动计算学期，新电子出现在对应轨道上。
5. 颜色延续 `timeline-organic.html` 的木质/自然暖色调，避免科技感青色。

### 问题排查与修复

#### 问题 1：页面空白，3D 场景不显示

**现象**：进入 `/growth-timeline` 后页面空白，仅显示底部提示文字。

**排查过程**：

1. 检查 DOM 发现 canvas 已创建且尺寸正确（1004×840）。
2. 检查渲染帧数发现 `requestAnimationFrame` 正常运行。
3. 通过 `renderer.setClearColor(0xff0000)` 测试，发现直接渲染能显示画面；恢复 `EffectComposer` 后再次空白。
4. 进一步隔离：仅保留 `RenderPass` 时画面正常，添加 `UnrealBloomPass` 后画面空白。

**根因**：`UnrealBloomPass` 在当前环境（Three.js r185 + 集成显卡/WebGL）下创建或执行异常，导致整个后处理管线输出透明画面。

**修复**：移除 `UnrealBloomPass`，仅保留 `EffectComposer + RenderPass`；电子和原子核的发光效果改用 `Sprite + AdditiveBlending` 与 `emissive` 材质实现。

#### 问题 2：页面被布局容器压缩

**现象**：3D 场景渲染后高度不足，被限制在内容区内。

**根因**：`DefaultLayout` 对内容区有 `padding`、`max-width` 和 `overflow` 约束，标准 `height: 100%` 无法占满可用空间。

**修复**：`.growth-timeline-page` 使用 `position: fixed`，占满 HeaderBar（56px）下方的整个内容区域，并在 `scene-wrapper` 使用 `position: absolute; inset: 0` 填满父容器。

## 五、2026-07-03 成长时间轴旋转与展开动效修复

### 修改原因

用户进一步反馈：

1. 电子看起来只是“挨着轨道边边上”，没有真正落在轨道中心。
2. 点击电子后没有实现「年轮逆旋转到电子所在位置，再从小变大铺满全屏」的效果，而是像直接跳转到对应位置。

根本原因是首屏中 SVG 轨道、原子核、电子轨道三者的旋转中心没有统一对齐，导致电子视觉偏移、点击热区错位，进而使点击后的旋转与展开动画无法按预期触发。本次修复将三者统一到同一个圆心，并强化展开动效的可靠性。

### 影响范围

- `src/features/growth-timeline/` 模块。
- 修改 `HeroRings.vue`、`ExperienceDetail.vue`。

### 修改范围

#### `src/features/growth-timeline/components/HeroRings.vue`

1. 统一旋转中心：
   - `.hero-rings__svg` 补充 `top: 50%; left: 50%; transform: translate(-50%, -50%);`，让 SVG 轨道以首屏中心为圆心。
   - `.hero-rings__nucleus` 补充同样的居中定位，使原子核与轨道、电子轨道完全同心。
2. 电子轨道容器已在此前修复中居中，现三者中心一致，电子精确位于对应学期轨道的中心线上。
3. 背景 faint rings 保持在系统旋转层之外，避免随旋转产生晃动。

#### `src/features/growth-timeline/components/ExperienceDetail.vue`

1. 移除 `<Transition name="detail-fade">` 包装层，改为 `v-show` 控制显隐，避免 Vue 过渡与 `layerStyle` 内联过渡在 `opacity`/`transform` 上冲突。
2. 打开动画增加双 `requestAnimationFrame` 强制回流，确保初始状态（宽高 0、缩放 0、透明度 0、圆形）先渲染，再进入最终状态（全屏、缩放 1、透明度 1、圆角 0），从而稳定触发「从小变大铺满全屏」的展开效果。
3. 保留关闭动画：详情层先收缩回展开原点，450ms 后再隐藏。

### 功能说明

## 十一、2026-07-03 成长时间轴并发回缩效果修复

### 修改原因

用户反馈：点击电子后「没看见并发回缩的效果」。

### 根因分析

1. **animation-delay 被内联样式覆盖**：`HeroRings.vue` 中每个环的内联 `animation-delay`（正向加载的错峰延迟）在添加 `.is-extinguished` 后仍然生效，导致外层环反而要等更久才启动，时序错乱。
2. **缺少真正的绘制/擦除效果**：`stroke-dasharray` 只设了一段长度等于周长的 dash，导致 `stroke-dashoffset` 变化时视觉上仍是整圈同时淡入/淡出，无法呈现「从一点开始回缩」的逆旋转感。
3. **详情提前遮挡动画**：`GrowthTimeline.vue` 中点击后 3000ms 就强制展开详情，而点击内层电子时回缩动画总时长约 3500ms，详情会在动画结束前弹出，遮挡回缩后半段。

### 影响范围

- `src/features/growth-timeline/components/HeroRings.vue`
- `src/features/growth-timeline/GrowthTimeline.vue`

### 修改范围

#### `src/features/growth-timeline/components/HeroRings.vue`

1. **让 is-extinguished 的动画立即启动**：
   - `.hero-rings__ring.is-extinguished` 增加 `animation-delay: 0s !important;`，覆盖内联正向加载延迟。

2. **实现真正的绘制与擦除**：
   - 将 `:stroke-dasharray` 从单段 `"ring.dasharray"` 改为 `"ring.dasharray ring.dasharray"`（dash + gap 各一段）。
   - 这样 `ringDrawIn` 从 `stroke-dashoffset: var(--ring-dasharray)` 到 `0` 会呈现线条从 3 点钟方向逐段绘制出来的效果。
   - `ringExtinguish` 从 `0` 到 `calc(var(--ring-dasharray) * -1)` 会呈现线条沿原路径被擦除的逆旋转效果。

3. **增强回缩视觉层次**：
   - `ringExtinguish` 关键帧增加 `stroke-width: 2.5 → 0` 与 `filter: drop-shadow(0 0 6px var(--ring-color)) → drop-shadow(0 0 0 transparent)`，让回缩过程更醒目。
   - 在环的内联样式中传入 `--ring-color`，供发光效果使用。

#### `src/features/growth-timeline/GrowthTimeline.vue`

1. **详情等待回缩完成后再展开**：
