# 2026-07-03 Tab 栏切换抖动综合修复

## 修改原因

顶部 NavTabs 与个人档案信息申报页面的模块切换仍存在明显抖动。前期修复仅优化了 NavTabs 宽度、ApplicationHub 面板高度和全局滚动条 gutter，但未触及结构性根因：

1. `DefaultLayout.vue` 中 `HeaderBar` 被 `<Transition mode="out-in">` + `:key="routeKey"` 驱动，每次路由变化整体卸载/重挂载，导致 NavTabs 闪烁。
2. `router-view` 包裹整页 `mode="out-in"` 过渡，内容区切换有 200ms 等待动画，与 tab 即时反馈冲突。
3. `useTabs.ts` 用 `fullPath` 作为 tab 唯一键，`/applications` 与 `/applications?tab=xxx` 被识别为不同 tab，每次切换模块都新增“个人档案信息申报”标签，引发 tab 栏宽度突变。
4. `ApplicationHub.vue` 同时渲染 10 个模块列表，首次渲染和模块切换时主线程压力大。
5. NavTabs tab 宽度按 6/9 阈值阶跃变化，跨越阈值时整体跳变。
6. 关闭按钮默认隐藏、hover 时才占位，挤压标题区域造成微抖动。
7. 缺少 `keep-alive`，切回旧 tab 时页面组件完全重建，`el-table` 等重新计算高度。

## 删除范围

- `src/app/layouts/DefaultLayout.vue`
  - 删除 `routeKey` computed（原用于驱动 HeaderBar 重挂载）。
  - 删除 `HeaderBar` 外层的 `<Transition name="layout-header" mode="out-in">`。
  - 删除 `router-view` 外层的 `<transition name="layout-page" mode="out-in">`。
  - 删除 `.layout-header-*`、`.layout-page-*` 动画类及响应式/无障碍中的对应引用。

- `src/app/layouts/components/NavTabs.vue`
  - 删除媒体查询中已不存在的 `.nav-tabs__nav` 样式规则。

- `src/features/applications/ApplicationHub.vue`
  - 删除 `.application-hub__panel-item` 上不再需要的 `opacity`/`transform`/`pointer-events`/`is-active`/`aria-hidden`/`inert` 等堆叠显隐样式。

## 新增范围

- `src/app/layouts/DefaultLayout.vue`
  - `HeaderBar` 改为直接挂载，不再因路由变化重渲染。
  - `router-view` 改用 `<keep-alive :max="20">` 缓存已访问页面。
  - `keep-alive` 的 `key` 根据 `routeSlot.meta?.stableTab` 选择 `path` 或 `fullPath`。

- `src/app/router/routes.ts`
  - ApplicationHub 路由 `meta` 增加 `stableTab: true`。

- `env.d.ts`
  - 补充 `vue-router` 的 `RouteMeta` 类型声明，增加 `stableTab?: boolean`。

- `src/shared/composables/useTabs.ts`
  - `handleRoute` 中按 `to.meta?.stableTab` 选择 `to.path` 或 `to.fullPath` 作为 tab 唯一键。

- `src/app/layouts/components/NavTabs.vue`
  - `activePath` 增加兜底同步逻辑。
  - tab 宽度公式改为平滑递减：`count <= 6 ? 168 : Math.max(112, 168 - (count - 6) * 8)`。
  - 关闭按钮改为绝对定位；标题预留 26px 空间；移动端/极小屏同步调整。
  - `.nav-tabs__surface` 增加 `contain: layout style`。

- `src/features/applications/ApplicationHub.vue`
  - 增加 `activeModule` 计算属性。
  - 面板从 `v-for` 渲染全部 10 个模块改为仅渲染当前激活模块。

## 影响范围

- 全局路由切换动画：移除了 HeaderBar 和页面内容的 out-in 过渡，tab 切换反馈更即时。
- ApplicationHub 模块切换：不再重复添加 tab，tab 数量稳定。
- ApplicationHub 性能：DOM 中仅保留当前模块，渲染压力降低。
- 已访问页面缓存：启用 `keep-alive` 后切回旧 tab 组件不复建，可能需要注意数据刷新逻辑（当前未引入业务副作用）。
- NavTabs 视觉：7 个及以上 tab 宽度比旧策略更窄，但变化更平滑。

## 验证结果

- 修改文件 ESLint 检查：无错误、无警告。
- `npx vite build`：构建成功。
- 浏览器验证（DevTools + 自动化快照）：
  - 切换“首页 / 成长时间轴 / 个人档案信息申报 / 个人档案信息 / 修改密码 / 职业规划 / 奖项总览”等 tab，HeaderBar 与 NavTabs 无闪烁。
  - 在 ApplicationHub 内切换模块（学科竞赛 → 创新创业 → 学术研究），URL 变化但 tab 栏不新增“个人档案信息申报”。
  - 7 个 tab 同时存在时宽度平滑，无整体跳变。
