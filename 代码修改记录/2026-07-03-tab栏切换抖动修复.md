## 一、删除记录

### 1.1 NavTabs.vue 中会触发重排的宽度动画

**涉及文件**：`src/app/layouts/components/NavTabs.vue`

**删除内容**：

- 计算属性 `tabWidthStyle` 中的 `--tab-width-active` 变量
- `.nav-tabs__item` 上 `transition: flex-basis ..., max-width ...`
- `.nav-tabs__item.is-active` 中对 `flex-basis`、`max-width`、`font-weight` 的修改
- `scrollIntoView` 中手动调用 `getBoundingClientRect` 与 `container.scrollBy` 的逻辑

**说明**：`flex-basis`/`max-width` 属于布局属性，过渡动画会导致浏览器每帧重排，是 tab 切换时相邻标签抖动的主因。

**删除内容**：

## 二、新增/修改记录

### 2.1 NavTabs.vue —— 固定宽度 + 合成层安全动画

**涉及文件**：`src/app/layouts/components/NavTabs.vue`

**修改内容**：

- 所有 tab 统一使用 `--tab-width` 固定宽度：`flex: 0 0 auto; width: var(--tab-width); min/max-width: var(--tab-width)`
- active 态仅保留颜色、背景、边框、阴影变化，并新增底部指示条（使用 `transform: scaleX` 动画）
- 删除 active 态的 `font-weight` 变化，避免文字度量重算
- 为 `.nav-tabs__item`、`.nav-tabs__surface` 统一设置 `box-sizing: border-box`
- `scrollIntoView` 改为 `requestAnimationFrame` 后调用原生 `el.scrollIntoView({ behavior: 'smooth', block: 'nearest', inline: 'nearest' })`，避免强制同步布局
- 保留 `prefers-reduced-motion: reduce` 规则
